## Design Context

### Users
명함 QR 코드를 스캔해 짧은 시간 안에 개발자의 역량을 가늠해보는 **스타트업·중견기업의 CTO와 기술 팀리드**.
보통 이동 중이거나 업무 틈새에 데스크톱·모바일로 열어보며, "이 사람을 더 알아볼 가치가 있나"를 1~2분 안에 판단하고 싶어한다.
필요한 정보는: 누구인가, 어떤 기술을 쓰는가, 대표 프로젝트가 무엇인가, 연락 수단.

### Brand Personality
**Confident · Editorial · Technically Literate.**
말투는 "─합니다" 체로 자신감 있고 전문가답게. 과장·겸손 모두 제거. 인터랙션과 디자인 자체가 실력 증명이 되어야 하므로, 사이트를 훑는 경험이 곧 기술 데모다.
감정적 목표: 신뢰감 · 차분한 자부심 · "이 사람 일하는 방식 보인다"는 체감.

### Aesthetic Direction
**Neo-minimalism / Editorial-brutalist hybrid.** 다크 테마 고정.
- 모노크로매틱 팔레트 + 단일 웜 오프화이트(#F2EDE4) 단일 액센트
- 에디토리얼 매거진 느낌: 섹션 번호(01/02/…), 헤어라인 디바이더, 모노스페이스 메타라벨
- 거대한 타이포그래피가 주인공 (clamp 기반 반응형, 굵기는 300-400 범위)
- 카드 박스를 지양하고 라인·그리드·여백으로 정보 위계 표현
- 장식적 효과(그라디언트 텍스트, 사이드-스트라이프 보더, 글로우, 글래스모피즘) 일절 금지

**Anti-references**: Dribbble-template 스타일의 사이안-네온 그라디언트 개발자 포트폴리오, 아이콘 + 라운디드 카드 그리드가 반복되는 템플릿형 사이트, 라이트모드 토글 / 이력서 PDF 다운로드 같은 일반화 기능.

### Design Principles
1. **Typography as structure** — 그리드와 여백이 뼈대지만, 시선을 잡는 건 타이포 스케일과 행간이다. 제목은 과감하게 키우고, 위계는 크기·굵기·거리만으로 표현.
2. **Editorial over template** — 동일한 카드 그리드를 반복하지 않는다. 섹션마다 레이아웃 접근을 바꾸되 같은 12컬럼 기반·같은 스페이싱 스케일·같은 타이포 스케일 위에서 변주한다.
3. **Hairline & negative space only** — 경계는 1px 헤어라인, 상태는 대비·투명도·위치로 표현. 배경 채움·그림자·라운디드 박스를 액센트로 쓰지 않는다.
4. **One monochromatic accent, used sparingly** — 액센트는 상호작용의 "현재"를 표시할 때만 등장. 60% 중립 서피스 · 30% 보조 텍스트/보더 · 10% 전경 화이트의 가중치를 지킨다.
5. **Interaction as proof** — 모든 상호작용(hover, transition, scroll progress, custom cursor)은 연출이 아니라 주장이다. 불필요한 애니메이션·바운스 이징 금지, 120-350ms · ease-out 기반 서브틀 모션만 허용.
6. **Accessibility is not optional** — `prefers-reduced-motion` 존중, 키보드 포커스 링 유지, 본문 대비 WCAG AA 이상.

### Technical Constraints
- Vite + React 18 + TypeScript (strict), Tailwind v4, Framer Motion, React Router v7
- GitHub Pages 정적 배포 (SSR 없음, Next.js 금지)
- 다크모드 고정, `localStorage`/`sessionStorage` 사용 금지
- 상태관리 라이브러리 미도입 (Redux/Zustand 등)

### 4pt Spacing Scale (this project)
`4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128 · 160`
섹션 상하 패딩은 `py-32 md:py-40` 이상(128–160) 관대하게, 관련 요소 그룹 내부는 8–16px 타이트하게.
